## コミット

### フォーマット

```
<emoji> <prefix>: <subject>

<body: 変更の理由を説明>
```

### emoji + prefix 一覧

| emoji | prefix   | 用途                               |
| ----- | -------- | ---------------------------------- |
| ✨    | feat     | 新機能の追加                       |
| 🐛    | fix      | バグ修正                           |
| ⚡️    | perf     | パフォーマンス改善                 |
| ♻️    | refactor | リファクタリング                   |
| 🔥    | remove   | コードやファイルの削除             |
| 💄    | style    | UIやスタイルの更新                 |
| 🚸    | ux       | UX・ユーザビリティの改善           |
| ♿️    | a11y     | アクセシビリティの改善             |
| 📝    | docs     | ドキュメントの追加・更新           |
| 💡    | comment  | ソースコード内コメントの追加・更新 |
| ✏️    | typo     | タイポ修正                         |
| 🤡    | mock     | モックの作成                       |
| 🏷️    | types    | 型の追加・更新                     |
| ✅    | test     | テストの追加・更新                 |
| 🔧    | config   | 設定ファイルの変更                 |

### ルール

- **subject**: 変更を簡潔に50文字以内で記述
- **body**: 変更の「Why（理由）」を説明する。差分をなぞるだけでなく背景・文脈・意図を書く
- **bodyの各文は改行で区切る（1文1行）**
- **コミットメッセージはすべて日本語で書く**
- **1コミット = 1つの論理的な変更。依存する変更は同じコミットにまとめる**

## ブランチ

feature/[チケット番号]形式
